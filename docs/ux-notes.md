# UX Notes

## Purpose

This document captures practical UX guidance for the Corehouse Pilates Studio website during the early project phase.

The goal is to make the website conversion-friendly, easy to scan, responsive, and easy to evolve as requirements become clearer.

## Brand and tone

The experience should feel:

- calm
- premium
- modern
- trustworthy
- welcoming
- clear

Avoid:

- noisy startup-style visuals
- aggressive marketing language
- confusing layouts
- over-designed decorative sections
- generic AI-looking website patterns

## Primary user goals

A visitor should quickly be able to:

- understand what Corehouse Pilates Studio offers
- feel trust in the studio
- preview classes or services
- learn about instructors
- view pricing or introductory options
- contact the studio or take the next step

## Primary conversion actions

The UX should consistently support one or more of these actions:

- book an intro class
- contact the studio
- explore classes
- view pricing

## Navigation notes

Navigation should be easy to rework later.

Important:

- keep navigation structure separate from navigation presentation
- support future movement from top navigation to sidebar-like patterns if needed
- keep mobile navigation simple and touch-friendly
- do not tightly couple page content to one navigation layout assumption

## Homepage UX notes

The homepage should:

- establish trust quickly
- explain the core value proposition clearly
- preview the most important parts of the business
- make the next action obvious

Recommended homepage flow:

1. Hero with clear message and main CTA
2. Benefits / value proposition
3. Classes preview
4. Instructors preview
5. Testimonials / trust signals
6. Final CTA

## Page UX notes

### About / Studio

Should communicate identity, atmosphere, and philosophy without becoming overly long.

### Classes

Should help visitors understand options quickly and reduce confusion about where to start.

### Pricing

Should present offers clearly and honestly.
The page should feel reassuring, not pushy.

### Instructors

Should feel human and credible.
The page should reinforce trust and professionalism.

### Contact

Should reduce friction.
Users should immediately understand how to reach or visit the studio.

### FAQ

Should reduce hesitation and answer practical beginner questions.

### Blog

Not phase-one critical, but the eventual UX should support educational discovery and SEO growth.

## Mobile-first notes

The website must feel natural on a phone from the beginning.

Important checks:

- content stacks cleanly
- buttons are easy to tap
- navigation remains usable
- text stays readable
- images do not break layout
- sections remain scannable

## Content notes

Use believable placeholder content until the client provides final material.
Do not use lorem ipsum.
Do not invent real pricing or credentials unless confirmed.

## Trust-building notes

The UX should reinforce trust through:

- clear messaging
- simple structure
- instructor credibility
- practical information
- restrained design
- visible next steps

## Open UX questions

These should be revisited when more client detail arrives:

- exact visual identity and color direction
- final navigation style
- final photography/image direction
- whether booking is internal or external
- whether the homepage should emphasize intro offers, instructor credibility, or class variety most strongly

## Definition of success

The UX direction is successful if:

- the site is easy to understand quickly,
- the mobile experience is strong,
- the design can evolve without structural rewrites,
- and visitors can always identify the next useful action.
